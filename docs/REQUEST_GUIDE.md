# 提需求时怎么说更好定位

以后你可以按下面格式提需求，我能更快定位并减少误改其他页面。

## 推荐格式

```text
模块：交接班 / 任务 / 用户 / 系统设置 / 看板 / 个人信息
页面位置：新增弹窗 / 查询区 / 列表 / 导出 / 登录页
要改什么：字段、按钮、下拉框、排序、保存逻辑
期望结果：展示什么、保存什么、是否必填、是否可搜索
是否影响其他模块：不影响 / 同步影响任务清单 / 同步影响导出
```

## 示例

```text
模块：交接班
页面位置：历史交接班列表
要改什么：不要标题，显示关键词
期望结果：关键词可排序，空值显示 --
是否影响其他模块：不影响任务清单
```

```text
模块：任务
页面位置：新增任务弹窗
要改什么：结束时间不能早于开始时间
期望结果：前端提示，后端也校验
是否影响其他模块：不影响交接班
```

## 截图标注建议

- 如果截图里有多个区域，告诉我是“顶部查询区”“新增弹窗”“列表第几列”。
- 如果是下拉框问题，说明是哪个弹窗里的哪个下拉框。
- 如果是保存问题，说明点击哪个按钮、提示了什么、是否有附件。
